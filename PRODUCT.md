# Product

## Register

product

## Users

The primary user is the owner of this personal blog. They manage posts, profile details, and core site presentation from a browser without working directly in repository configuration files. Visitors remain readers and never receive write access.

## Product Purpose

Firefly combines a public personal blog with a lightweight, Git-backed authoring layer. Success means the owner can move from reading a live page to editing that exact article, publishing a new post, or changing safe site settings with very little context switching, while GitHub and Cloudflare continue to provide version history and deployment.

## Brand Personality

Calm, personal, and focused. Author tools should feel like a natural extension of the existing Firefly theme rather than a disconnected enterprise dashboard.

## Anti-references

- A generic admin dashboard that visually competes with the blog.
- Dense WordPress-style navigation for a single-author workflow.
- Editing controls shown permanently to ordinary visitors.
- Decorative motion or glass effects that distract from writing.

## Design Principles

- Start authoring from the content being read.
- Keep public reading and private writing visually connected but operationally separate.
- Let GitHub permissions remain the source of truth for write access.
- Prefer a small set of clear author actions over a large control surface.
- Preserve Markdown as the portable source format.

## Accessibility & Inclusion

Target WCAG 2.1 AA for controls and text, preserve keyboard access, provide visible focus states, and respect reduced-motion preferences. Author actions must remain usable on desktop and mobile.
